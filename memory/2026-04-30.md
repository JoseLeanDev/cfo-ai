- `40b6e09` - **CRITICAL:** Reverted all manual `isPostgres` conditionals in agents. Now ALL agents write pure SQLite and the translator in `connection.js` handles conversion. Fixed `julianday(a)-julianday(b)` → `(a::date - b::date)` instead of broken `EXTRACT(DAY FROM ...)`.

**Key learnings:**
1. Manual `if (isPostgres)` conditionals in agent files are ANTI-PATTERN — they duplicate code and conflict with the automatic translator wrapper.
2. `julianday(a) - julianday(b)` in SQLite = number of days. In PostgreSQL: `(a::date - b::date)` returns integer days directly. `EXTRACT(DAY FROM integer)` FAILS in PG.
3. `INSERT OR REPLACE` → `INSERT INTO` (simplified by translator). For upserts, use `DELETE + INSERT` for cross-engine compatibility.
4. The translator `sqliteToPostgres()` in `connection.js` is the SINGLE source of truth for SQL dialect conversion.

**Translator now handles:**
- `datetime('now')` → `NOW()`
- `date('now')` → `CURRENT_DATE`
- `date('now', '-X days')` → `CURRENT_DATE - INTERVAL 'X days'`
- `date('now', '-X months')` → `CURRENT_DATE - INTERVAL 'X months'`
- `date(?, '+X days')` → `?::date + INTERVAL 'X days'`
- `strftime('%Y-%m', col)` → `TO_CHAR(col::timestamp, 'YYYY-MM')`
- `julianday(a) - julianday(b)` → `(a::date - b::date)`
- `MAX(0, x)` → `GREATEST(0, x)`
- `?` → `$1, $2...`
- `INSERT OR REPLACE` → `INSERT INTO`
- `IFNULL` → `COALESCE`
