# 2026-04-24 — CFO AI Fixes & Chatbot Issues

## Chatbot Error 500 — CxP

**Error:** `column "proveedor" does not exist` al escribir "CxP" en el chatbot.

**Causa raíz:** Las queries del endpoint `/api/agents/chat` en `agents.js` usaban:
1. Funciones SQLite (`julianday`, `date('now')`) en la rama PostgreSQL
2. Nombres de columnas de SQLite (`proveedor`, `cliente`, `monto`) en PostgreSQL

**Columnas diferentes SQLite vs PostgreSQL:**
| Tabla | SQLite | PostgreSQL |
|-------|--------|------------|
| cuentas_cobrar | `cliente` | `cliente_nombre` |
| cuentas_cobrar | `monto` | `monto_pendiente` |
| cuentas_pagar | `proveedor` | `proveedor_nombre` |
| cuentas_pagar | `monto` | `monto_pendiente` |

**Fix aplicado (commit `265f2ed`):**
- CxP: `proveedor` → `proveedor_nombre`, `monto` → `monto_pendiente`
- CxC: `cliente` → `cliente_nombre`, `monto` → `monto_pendiente`
- SAT: `date('now')` → `CURRENT_DATE`
- Conciliación: `julianday` → `EXTRACT(DAY FROM ...)`
- CCC: `julianday` → `EXTRACT(DAY FROM ...)`

**Archivo:** `backend/src/routes/agents.js`

## Patrón de Problema Cruzado

Cada vez que se modifica un endpoint que usa queries SQL, verificar:
1. Funciones SQLite vs PostgreSQL (`julianday`, `date`, `datetime`)
2. Nombres de columnas (difieren entre SQLite y PostgreSQL)
3. Parámetros posicionales (`$1` vs `?`)

## PostgreSQL en Render
- La DB en Render usa PostgreSQL con columnas snake_case
- La DB local usa SQLite con columnas más simples
- Migración PostgreSQL: `backend/database/migrate-postgres.js`
- SQLite: `backend/database/migrate.js`
