## Debug: Cash Conversion Cycle (CCC) showing 0 values

**Problem**: Endpoint `/api/analisis/working-capital` returning all zeros for DSO, DPO, DIO, C2C

**Root causes found**:
1. SQL queries used PostgreSQL column names (`monto_pendiente`) but Render uses SQLite (`monto`)
2. PostgreSQL queries used SQLite functions (`julianday()`, `date()`, `strftime()`)
3. PostgreSQL params used `?` instead of `$1`
4. No proper fallback when no data exists in database

## Technical notes

**PostgreSQL vs SQLite differences**:
- Parameter placeholders: `$1` (PG) vs `?` (SQLite)
- Date functions: `EXTRACT(DAY FROM...)` (PG) vs `julianday()` (SQLite)
- String formatting: `TO_CHAR()` (PG) vs `strftime()` (SQLite)
- Type casting: `::numeric`, `::integer` (PG) vs implicit (SQLite)
- Current date: `CURRENT_DATE` (PG) vs `date('now')` (SQLite)
