#### Compatibilidad PostgreSQL vs SQLite
Los agentes usan el wrapper de `database/connection.js` que automáticamente:
- Detecta si es PostgreSQL (producción/Render) o SQLite (local)
- Convierte sintaxis: `datetime('now')` → `NOW()`, `?` → `$1`, `$2`, etc.
- Usa SSL para PostgreSQL en Render

#### Correcciones de schema
Las tablas en SQLite local NO tienen `empresa_id`, así que se corrigieron los INSERTS:
- `snapshots_diarios` → sin empresa_id
- `snapshots_financieros` → sin empresa_id
- `alertas_financieras` → usa `titulo` en vez de `mensaje`
- `briefings_diarios` → usa `resumen_ejecutivo` + `insights_json`
- `cierres_mensuales` → usa `mes` + `notas`

En PostgreSQL (Render) probablemente sí tienen `empresa_id`, pero los queries actuales funcionan porque las columnas extra son ignoradas o se manejan por la app.
