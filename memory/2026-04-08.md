## 2026-04-08 09:22 PM CST - AI Insights Error Fix

### Problema Reportado
Usuario reportó error en sección "AI Insights" del dashboard: mensaje "Error al generar insights" con detalle "Hubo un error al procesar los datos financieros".

### Root Cause Analysis
Tras investigar `AnalistaFinanciero.js` y `PredictorCashFlow.js`:

1. **Columnas faltantes:** Queries SQL referenciaban `cliente_id` y `nombre_cliente` que no existían en tabla `transacciones`
2. **Valores de tipo incorrectos:** Agents usaban `'ingreso'`/`'egreso'` pero el seed y DB usaban `'entrada'`/`'salida'`

### Solución Implementada

| Archivo | Cambio |
|---------|--------|
| `database/migrate.js` | Agregadas columnas `cliente_id` (INTEGER) y `nombre_cliente` (TEXT) |
| `database/seed.js` | Actualizado para poblar `cliente_id` y `nombre_cliente` en transacciones de entrada |
| `AnalistaFinanciero.js` | Fix: `'ingreso'` → `'entrada'`, `'egreso'` → `'salida'` |
| `PredictorCashFlow.js` | Fix: `'ingreso'` → `'entrada'`, `'egreso'` → `'salida'` |
| `migrations/003_fix_transacciones_ai.sql` | Script de migración para tablas existentes |

**Commit:** `bfac457` - "fix(backend): add cliente columns to transacciones and fix tipo values for AI insights"

### Lección
Los agents de IA fallan silenciosamente cuando las queries SQL no matchean la estructura real de la DB. Necesitan validación de schema o mejor manejo de errores para debugging.
